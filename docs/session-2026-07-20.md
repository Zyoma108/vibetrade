# Strategy Optimization — July 2026 Session

## What was done

Audited the trading bot database for the losing week 13.07-20.07, ran parameter sweeps
across all 3 historical databases, tested configurations V1 and V2, arrived at an
optimal config that improves win rate by 9-25% across all periods.

## Key findings

1. **vol_mult=5.0 is optimal** — 60% WR, +$16.65 on good period.
   At 6.5: near break-even. At 8.0: guaranteed loss.
2. **dump_volume_mult=3.0 HURTS** — disabled it. Sweep showed dump=0 better.
3. **SL=5% + RR=2.0** gives best win rate — TP at +10% is reachable.
   V1 used RR=2.5 → TP at +12.5% → too far, TP hits dropped 60-100%.
4. **risk=0.6% + partial=35%** most stable. Can scale risk to 1.0% for more PnL.
5. **Circuit Breaker enabled** — two losing streaks of 5 and 7 were unprotected.
6. **15 US-stock tokens** (AAPL, COIN, HOOD, MSTR, etc.) added to exclude_coins —
   they require ByBit agreement that isn't signed.

## V2 Config (current)

| Parameter | Value | Why |
|---|---|---|
| volume_surge_mult | 5.0 | Critical — breaks above 6.5 |
| stop_loss_pct | 5.0 | Protection from noise |
| risk_reward_ratio | 2.0 | TP = +10% (reachable) |
| risk_per_trade_pct | 0.6 | Conservative, scale to taste |
| dump_volume_mult | 0.0 | Off — hurts results |
| partial_close_pct | 35.0 | Earlier lock-in |
| pre_surge_max_pct | 8.0 | Soft pre-pump filter |
| circuit_breaker_enabled | true | Protection from streaks |
